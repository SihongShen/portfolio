Muryo Memo grew out of a real problem I kept running into as a comic con regular. I make my own merchandise to exchange with other fans at events, but I never had a good way to track what I'd made, what I still needed to prepare, and who I needed to find on the day. Computer folders aren't portable, plain memos get messy, and tools like Notion didn't fit how I wanted to work — so I built a small, focused app that does exactly one thing: keep track of freebies and the people you need to exchange them with.  

On the technical side, the project uses a React frontend paired with an Express and MongoDB backend for basic product management via a REST API. The background is a physics-driven particle animation built with p5.js and matter-js, running in instance mode and modularized under a single initializer that exposes a small API for palette switching and cleanup. Color state is shared between the sketch and React components through a ColorContext, and the whole project is deployed on Digital Ocean.
